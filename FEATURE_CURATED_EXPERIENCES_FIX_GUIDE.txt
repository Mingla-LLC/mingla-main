ROOT CAUSE SUMMARY (read this first)
-------------------------------------
There are 3 stacked bugs preventing curated cards from appearing:

BUG 1 (CRITICAL): PreferencesSheet saves categories but drops intents.
  - User selects "Solo Adventure" intent
  - On Apply, only selectedCategories (e.g. ['nature']) are saved to DB/cache
  - selectedIntents (e.g. ['solo-adventure']) are silently lost
  - RecommendationsContext looks for intents inside categories — finds none
  - All useCuratedExperiences hooks have enabled: false — no API call ever made

BUG 2 (SECONDARY): Curated cards gated behind non-zero regular cards.
  - Even if Bug 1 is fixed, the useEffect that syncs recommendations to state
    only runs setRecommendations if recommendationsData.length > 0
  - If regular cards return 0, curated cards are never set and deck is empty

BUG 3 (TERTIARY): interleaveCards drops curated cards past end of regular deck.
  - Logic: insert 1 curated after every 3 regular
  - If 0-2 regular cards: 0 curated cards shown
  - If 6 regular cards: only 2 curated shown, rest dropped
  - User expects up to 20 curated cards in the deck

STEP 2: Fix interleaveCards function (Bug 3 — TERTIARY)
--------------------------------------------------------
File: app-mobile/src/contexts/RecommendationsContext.tsx
Location: lines 111-125 (the interleaveCards function near the top of the file)

Replace the entire function:

    function interleaveCards(
      regular: Recommendation[],
      curated: Recommendation[]
    ): Recommendation[] {
      if (curated.length === 0) return regular;
      const result: Recommendation[] = [];
      let curatedIdx = 0;
      regular.forEach((card, idx) => {
        result.push(card);
        if ((idx + 1) % 3 === 0 && curatedIdx < curated.length) {
          result.push(curated[curatedIdx++]);
        }
      });
      return result;
    }

With this updated version:

    function interleaveCards(
      regular: Recommendation[],
      curated: Recommendation[]
    ): Recommendation[] {
      if (curated.length === 0) return regular;
      if (regular.length === 0) return curated;
      const result: Recommendation[] = [];
      let curatedIdx = 0;
      regular.forEach((card, idx) => {
        result.push(card);
        if ((idx + 1) % 3 === 0 && curatedIdx < curated.length) {
          result.push(curated[curatedIdx++]);
        }
      });
      while (curatedIdx < curated.length) {
        result.push(curated[curatedIdx++]);
      }
      return result;
    }

What changed:
  - Added: `if (regular.length === 0) return curated;`
    → Allows curated-only deck when no regular cards exist
  - Added: `while (curatedIdx < curated.length) { ... }` at the end
    → Appends all remaining curated cards after the regular deck ends

COMMON MISTAKES TO AVOID
--------------------------
1. DO NOT change the edge function. generate-curated-experiences/index.ts
   is correct. The pairings, PAIRINGS_BY_TYPE keys, and Places API calls
   are all working.

2. DO NOT change useCuratedExperiences.ts. The enabled logic there is:
     enabled: enabled && location !== null
   This is correct — it delegates to the caller (RecommendationsContext).

3. DO NOT change the LOAD logic in PreferencesSheet.tsx (lines 483-495).
   The separation of intents and categories on load is already correct:
     if (intentIds.has(item)) loadedIntents.push(item);
     else loadedCategories.push(item);
   Only the SAVE path (handleApplyPreferences) needs to be fixed.

4. The edge function default for experienceType is 'solo_adventure' (underscored)
   but PAIRINGS_BY_TYPE keys use hyphens. This is harmless because the hook
   always passes the correct hyphenated value — but don't rely on the default.

5. Location must be non-null for the hooks to fire (the hook checks
   `enabled && location !== null`). If the user hasn't granted location
   permission or search location is empty, curated cards won't load.
   Make sure you have a location set in preferences before testing.
