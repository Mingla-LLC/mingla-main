RECOMMENDATION TYPE UNIFICATION — IMPLEMENTATION GUIDE
=======================================================

WHAT THIS FIXES:
Two copies of the Recommendation interface exist with different shapes.
CardsCacheContext has a narrow version (openingHours: string, missing website/phone/placeId/strollData).
RecommendationsContext has the full version.
This causes silent type mismatches when cards flow through the cache.

APPROACH:
Extract the full Recommendation interface to a shared types file.
Both contexts import from it. RecommendationsContext re-exports it.
Zero changes to any consumer files. Zero runtime changes.

TOTAL FILES CHANGED: 3 (1 new, 2 modified)
ESTIMATED TIME: 10 minutes

=======================================================

STEP 1: Create the canonical types file
=======================================================

File: app-mobile/src/types/recommendation.ts

Create this file with the FULL Recommendation interface — copy it EXACTLY
from RecommendationsContext.tsx lines 31-109. This is the version with:
- openingHours: string | { open_now?: boolean; weekday_text?: string[] } | null
- website?: string | null
- phone?: string | null
- placeId?: string
- strollData?: { anchor, companionStops, route, timeline }

Export it as: export interface Recommendation { ... }

Add a brief doc comment at the top explaining this is the single source of
truth for the Recommendation type.

DO NOT add any other types to this file — just Recommendation.


STEP 2: Update CardsCacheContext.tsx
=======================================================

c) DO NOT change anything else in this file. The CardsCacheEntry interface
   on line 51 already references Recommendation by name, and setCachedCards

COMMON MISTAKES TO AVOID
=======================================================

1. DO NOT import Recommendation in CardsCacheContext from RecommendationsContext.
   This creates a circular dependency (RecommendationsContext already imports
   useCardsCache from CardsCacheContext).

2. DO NOT forget the re-export in RecommendationsContext:
   export type { Recommendation };
   Without this, every file importing Recommendation from RecommendationsContext breaks.

3. DO NOT change any consumer files. The re-export ensures they keep working.

4. DO NOT use `export { Recommendation }` (value export) — use
   `export type { Recommendation }` (type-only export) since it's a type.

5. DO NOT modify the Recommendation shape. This fix is ONLY about
   deduplication — the interface content should be identical to what
   RecommendationsContext currently has.
