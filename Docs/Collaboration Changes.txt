## 🎯 What Collaboration Sessions Are

**Collaboration Sessions** (also called "Board Sessions" or "Boards") allow **groups of friends to plan experiences together** in real-time.

### Core Concept:
- Multiple users swipe, save, vote, and discuss experiences together
- Each board has its own set of saved cards, preferences, and chat
- Real-time synchronization keeps everyone in sync
- Users can switch between "Solo Mode" and multiple board sessions

### Example Use Cases:
- **Date Night Planning:** Couple plans romantic dinner + activity
- **Group Hangout:** 5 friends find something to do this Saturday
- **Weekend Trip:** Friends collaborate on a multi-day itinerary
- **Fitness Squad:** Regular workout buddies find new classes

## 📜 How It Was Before (Original Concept)

### **Vision: Full Real-Time Collaborative Experience Planning**

The original design document (`COLLABORATION_FEATURE_PROMPT.md`) outlined a comprehensive system:

#### ✅ **What Was Planned:**

1. **Session Types**
   - Date Night
   - Group Hangout  
   - Weekend Trip
   - Special Occasion
   - Custom sessions

2. **Collaborative Preferences**
   - Each board has its own preferences (budget, categories, vibes)
   - Participants vote on preferences
   - Cards generated based on board preferences, not individual ones
   - Preference conflicts resolved by voting

3. **Real-Time Board Features**
   - **Saved Cards Tab:** All cards saved by any participant
   - **Voting System:** Vote yes/no on each card
   - **RSVP System:** Indicate attendance for specific cards
   - **Discussion Tab:** Chat about experiences with @mentions
   - **Card-Specific Threads:** Comment on individual cards

4. **Real-Time Synchronization**
   - WebSocket connections
   - When someone saves a card → appears for everyone instantly
   - When someone votes → count updates in real-time
   - When someone chats → message appears immediately
   - Typing indicators
   - Presence indicators (who's online)

5. **Session Management**
   - Create sessions with name + friends
   - Send invitations
   - Accept/decline invites
   - Admin controls (rename, manage participants, delete)
   - Leave sessions
   - Archive sessions
   - Session states: Active, Inactive, Archived, Deleted

6. **Notifications**
   - Invite notifications
   - Activity notifications (saves, votes, RSVPs)
   - Mention notifications
   - RSVP reminders
   - Push notifications + in-app toasts

7. **Board-Aware Saved Cards**
   - Cards saved from boards have badges showing which board
   - From personal Saved tab, can navigate back to board
   - See board context (votes, RSVPs, discussion)

8. **Advanced Features**
   - Offline queueing with sync when back online
   - Conflict resolution for simultaneous actions
   - Session analytics
   - Transfer admin rights
   - Auto-delete inactive sessions after 7 days

## 🔄 How It Is Now (Current Implementation)

### **Reality: Partial Implementation with Mock Data**

The current codebase has the **UI shell** built but lacks **backend integration and real-time functionality**.

#### ✅ **What IS Implemented:**

1. **CollaborationModule Component** (`/components/CollaborationModule.tsx`)
   - 3-tab modal interface
   - **Sessions Tab:** View active sessions, switch modes
   - **Invites Tab:** View sent/received invites
   - **Create Tab:** 3-step wizard to create new sessions

2. **Session Creation Flow**
   - Step 1: Enter session name
   - Step 2: Select friends (multi-select with pre-selection support)
   - Step 3: Confirm and create
   - Creates local session object
   - Switches user to board mode

3. **CollaborationBoard Component** (`/components/collaboration/CollaborationBoard.tsx`)
   - Full board view with 2 tabs:
     - **Saved Cards:** Grid of saved experiences
     - **Discussion:** Chat interface
   - Board header with participants
   - Settings menu (rename, manage, mute, leave, delete)
   - Admin-only controls

4. **UI Features Working:**
   - Card voting UI (yes/no buttons)
   - RSVP UI (yes/no/maybe)
   - Message composition with @mentions
   - Participant avatars
   - Session status badges
   - Pre-selected friend support
   - Modal navigation
   - Coach mark integration

5. **State Management:**
   - Local React state for sessions
   - Props passed from App.tsx
   - Mode switching (solo vs board ID)
   - Session data stored in `boardsSessions` array

#### ❌ **What is NOT Implemented:**

1. **No Backend Integration**
   - All data is mock/hardcoded (`MOCK_ACTIVE_SESSIONS`, `MOCK_FRIENDS`)
   - Sessions not persisted to database
   - No API calls
   - No real user data

2. **No Real-Time Sync**
   - No WebSocket connections
   - No real-time updates
   - Changes don't propagate to other users
   - No presence indicators
   - No typing indicators

3. **No Actual Voting/RSVP Logic**
   - Vote buttons render but don't affect anything
   - RSVP buttons render but don't affect anything
   - Counts are hardcoded
   - No vote persistence
   - No RSVP tracking

4. **No Collaborative Preferences**
   - Boards don't have their own preferences
   - No preference voting
   - Cards still generate from user's personal preferences
   - No way to set board budget, categories, etc.

5. **No Invitation System**
   - Invites are displayed but not sent
   - Accept/decline buttons don't work
   - No notifications
   - No invite expiration
   - Can't actually invite real friends

6. **No Discussion Functionality**
   - Chat UI renders but messages aren't sent
   - @mentions render but don't notify
   - No message persistence
   - No card-specific threads
   - No real message history

7. **No Session Persistence**
   - Sessions reset on page refresh
   - No saved sessions in database
   - No session history
   - No way to rejoin previous sessions

8. **No Board-Specific Cards**
   - Saved cards don't show which board they came from
   - No board badges on saved cards
   - Can't navigate from personal saved to board
   - No board context preserved

9. **No Notifications**
   - No push notifications
   - No in-app toasts
   - No notification badges
   - No notification center

10. **No Offline Support**
    - No queuing
    - No sync when back online
    - Breaks if network fails

## 🏗️ Technical Architecture

### **Database Schema (Supabase)**

#### `collaboration_sessions`
```sql
CREATE TABLE collaboration_sessions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  type TEXT, -- 'date-night', 'group-hangout', etc.
  status TEXT DEFAULT 'active', -- 'active', 'inactive', 'archived', 'deleted'
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  last_activity TIMESTAMPTZ DEFAULT NOW(),
  metadata JSONB -- extra settings
);
```

#### `collaboration_participants`
```sql
CREATE TABLE collaboration_participants (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  session_id UUID REFERENCES collaboration_sessions(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id),
  role TEXT DEFAULT 'participant', -- 'admin', 'participant'
  status TEXT DEFAULT 'active', -- 'active', 'left', 'removed'
  joined_at TIMESTAMPTZ DEFAULT NOW(),
  notifications_muted BOOLEAN DEFAULT FALSE,
  UNIQUE(session_id, user_id)
);
```

#### `collaboration_invites`
```sql
CREATE TABLE collaboration_invites (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  session_id UUID REFERENCES collaboration_sessions(id) ON DELETE CASCADE,
  from_user_id UUID REFERENCES users(id),
  to_user_id UUID REFERENCES users(id),
  status TEXT DEFAULT 'pending', -- 'pending', 'accepted', 'declined', 'canceled'
  created_at TIMESTAMPTZ DEFAULT NOW(),
  expires_at TIMESTAMPTZ DEFAULT (NOW() + INTERVAL '7 days'),
  responded_at TIMESTAMPTZ,
  UNIQUE(session_id, to_user_id)
);
```

#### `collaboration_saved_cards`
```sql
CREATE TABLE collaboration_saved_cards (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  session_id UUID REFERENCES collaboration_sessions(id) ON DELETE CASCADE,
  card_data JSONB NOT NULL, -- Full card object
  saved_by UUID REFERENCES users(id),
  saved_at TIMESTAMPTZ DEFAULT NOW(),
  is_locked BOOLEAN DEFAULT FALSE, -- locked for decision
  locked_at TIMESTAMPTZ
);
```

#### `collaboration_votes`
```sql
CREATE TABLE collaboration_votes (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  session_id UUID REFERENCES collaboration_sessions(id) ON DELETE CASCADE,
  card_id UUID REFERENCES collaboration_saved_cards(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id),
  vote TEXT NOT NULL, -- 'yes', 'no'
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(card_id, user_id)
);
```

#### `collaboration_rsvps`
```sql
CREATE TABLE collaboration_rsvps (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  session_id UUID REFERENCES collaboration_sessions(id) ON DELETE CASCADE,
  card_id UUID REFERENCES collaboration_saved_cards(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id),
  status TEXT NOT NULL, -- 'yes', 'no', 'maybe'
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(card_id, user_id)
);
```

#### `collaboration_messages`
```sql
CREATE TABLE collaboration_messages (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  session_id UUID REFERENCES collaboration_sessions(id) ON DELETE CASCADE,
  card_id UUID REFERENCES collaboration_saved_cards(id), -- NULL for general chat
  user_id UUID REFERENCES users(id),
  message TEXT NOT NULL,
  mentions UUID[], -- Array of mentioned user IDs
  created_at TIMESTAMPTZ DEFAULT NOW(),
  edited_at TIMESTAMPTZ,
  deleted_at TIMESTAMPTZ
);
```

#### `collaboration_preferences`
```sql
CREATE TABLE collaboration_preferences (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  session_id UUID REFERENCES collaboration_sessions(id) ON DELETE CASCADE,
  preferences JSONB NOT NULL, -- Budget, categories, vibes, time, location
  set_by UUID REFERENCES users(id),
  set_at TIMESTAMPTZ DEFAULT NOW()
);
```

### **Real-Time Subscriptions (Supabase)**

```typescript
// Subscribe to session changes
const sessionChannel = supabase
  .channel(`session:${sessionId}`)
  .on(
    'postgres_changes',
    {
      event: '*',
      schema: 'public',
      table: 'collaboration_saved_cards',
      filter: `session_id=eq.${sessionId}`
    },
    (payload) => {
      // New card saved, update UI
      handleCardSaved(payload.new);
    }
  )
  .on(
    'postgres_changes',
    {
      event: '*',
      schema: 'public',
      table: 'collaboration_votes',
      filter: `session_id=eq.${sessionId}`
    },
    (payload) => {
      // New vote, update counts
      handleVoteChange(payload);
    }
  )
  .on(
    'postgres_changes',
    {
      event: 'INSERT',
      schema: 'public',
      table: 'collaboration_messages',
      filter: `session_id=eq.${sessionId}`
    },
    (payload) => {
      // New message, append to chat
      handleNewMessage(payload.new);
    }
  )
  .subscribe();
```

### **API Functions**

```typescript
// Create session
async function createSession(name: string, type: string, friendIds: string[]) {
  const { data: session } = await supabase
    .from('collaboration_sessions')
    .insert({ name, type, created_by: currentUser.id })
    .select()
    .single();
  
  // Add creator as admin participant
  await supabase
    .from('collaboration_participants')
    .insert({ 
      session_id: session.id, 
      user_id: currentUser.id, 
      role: 'admin' 
    });
  
  // Send invites to friends
  const invites = friendIds.map(friendId => ({
    session_id: session.id,
    from_user_id: currentUser.id,
    to_user_id: friendId
  }));
  
  await supabase
    .from('collaboration_invites')
    .insert(invites);
  
  return session;
}

// Vote on card
async function voteOnCard(cardId: string, vote: 'yes' | 'no') {
  // Upsert vote (insert or update)
  await supabase
    .from('collaboration_votes')
    .upsert({ 
      card_id: cardId, 
      user_id: currentUser.id, 
      vote 
    }, {
      onConflict: 'card_id,user_id'
    });
  
  // Real-time broadcast happens automatically
}

// RSVP to card
async function rsvpToCard(cardId: string, status: 'yes' | 'no' | 'maybe') {
  await supabase
    .from('collaboration_rsvps')
    .upsert({ 
      card_id: cardId, 
      user_id: currentUser.id, 
      status 
    }, {
      onConflict: 'card_id,user_id'
    });
}

// Send message
async function sendMessage(sessionId: string, message: string, cardId?: string) {
  // Parse @mentions from message
  const mentions = parseMentions(message);
  
  const { data } = await supabase
    .from('collaboration_messages')
    .insert({
      session_id: sessionId,
      card_id: cardId,
      user_id: currentUser.id,
      message,
      mentions
    })
    .select()
    .single();
  
  // Send notifications to mentioned users
  for (const mentionedUserId of mentions) {
    await sendNotification(mentionedUserId, 'mention', data);
  }
  
  return data;
}
```
