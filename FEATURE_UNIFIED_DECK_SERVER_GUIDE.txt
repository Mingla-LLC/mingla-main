ROUTING LOGIC (inside the handler):

  A. Parse request body — all fields from ServeDeckRequest (see spec)

  B. Determine deckMode:
     const { intents, categories } = separateIntentsAndCategories(body.categories);
     const deckMode = isNatureMode(categories) ? 'nature' : 'curated';

  C. If warmPool → warm the right pool:
     - nature: call nature search + store in pool (same as discover-nature warmPool)
     - curated: call curated pool warming for each intent

  D. If deckMode === 'nature':
     —— Copy the EXACT logic from discover-nature/index.ts lines 300-553 ——
     The flow:
     1. Calculate maxDistKm from travel constraint
     2. Try pool-first (serveCardsFromPipeline with category='Nature')
     3. If pool insufficient: batchSearchPlaces with 8 NATURE_TYPES
     4. Deduplicate, filter by distance, budget, datetime
     5. Sort by rating + reviews
     6. Offset pagination (batchSeed * limit)
     7. Generate AI descriptions (same OpenAI batch call)
     8. Build NatureDeckCard[] with cardType: 'nature'
     9. Store in card_pool (fire-and-forget)
     10. Return

  E. If deckMode === 'curated':
     —— Copy the EXACT logic from generate-curated-experiences/index.ts ——
     The flow:
     1. Determine experience types: intents.length > 0 ? intents : ['solo-adventure']
     2. Try pool-first for EACH type (serveCuratedCardsFromPool)
     3. If pool insufficient: run the full pipeline per type
        - Fetch places by super-categories
        - Build triads or single-stop cards
        - Generate stop descriptions via OpenAI
     4. Merge all types, shuffle with seededShuffle
     5. Offset pagination (batchSeed * limit)
     6. Build CuratedDeckCard[] with cardType: 'curated'
     7. Store in card_pool (fire-and-forget)
     8. Return

  F. Build unified response:
     { cards, deckMode, total, meta: { fromPool, fromApi, elapsed } }

```typescript
import { supabase } from './supabase';

export interface DeckParams {
  location: { lat: number; lng: number };
  categories: string[];
  budgetMin: number;
  budgetMax: number;
  travelMode: string;
  travelConstraintType: 'time' | 'distance';
  travelConstraintValue: number;
  datetimePref?: string;
  dateOption?: string;
  timeSlot?: string | null;
  batchSeed?: number;
  limit?: number;
}

export interface DeckResponse {
  cards: any[];  // Raw server response — mapped to Recommendation in hook
  deckMode: 'nature' | 'curated';
  total: number;
  meta: { fromPool: number; fromApi: number; elapsed: number };
}

class DeckService {
  async fetchDeck(params: DeckParams): Promise<DeckResponse> {
    const { data, error } = await supabase.functions.invoke('serve-deck', {
      body: params,
    });
    if (error) throw error;
    return data as DeckResponse;
  }

  async warmDeckPool(params: Omit<DeckParams, 'limit' | 'batchSeed'>): Promise<void> {
    try {
      await supabase.functions.invoke('serve-deck', {
        body: { ...params, warmPool: true, limit: 40 },
      });
    } catch {
      // Silent — non-critical background operation
    }
  }
}

export const deckService = new DeckService();
```

```typescript
import { useQuery } from '@tanstack/react-query';
import { deckService, DeckParams, DeckResponse } from '../services/deckService';
import type { Recommendation } from '../contexts/RecommendationsContext';

// Import converters (move from RecommendationsContext to a shared location)
// For now, import from RecommendationsContext (we'll move them in Step 7)

interface UseDeckCardsParams {
  location: { lat: number; lng: number } | null;
  categories: string[];
  budgetMin: number;
  budgetMax: number;
  travelMode: string;
  travelConstraintType: 'time' | 'distance';
  travelConstraintValue: number;
  datetimePref?: string;
  dateOption?: string;
  timeSlot?: string | null;
  batchSeed: number;
  enabled: boolean;
}

export function useDeckCards(params: UseDeckCardsParams): {
  cards: Recommendation[];
  deckMode: 'nature' | 'curated';
  isLoading: boolean;
  isFetching: boolean;
  isFullBatchLoaded: boolean;
  error: Error | null;
  refetch: () => void;
} {
  const { location, enabled, ...restParams } = params;

  const query = useQuery<DeckResponse>({
    queryKey: [
      'deck-cards',
      location?.lat,
      location?.lng,
      params.categories.sort().join(','),
      params.budgetMin,
      params.budgetMax,
      params.travelMode,
      params.travelConstraintType,
      params.travelConstraintValue,
      params.datetimePref,
      params.dateOption ?? 'now',
      params.timeSlot ?? '',
      params.batchSeed,
    ],
    queryFn: () =>
      deckService.fetchDeck({
        ...restParams,
        location: location!,
        limit: 20,
      }),
    staleTime: 30 * 60 * 1000,     // 30 minutes
    gcTime: 2 * 60 * 60 * 1000,    // 2 hours
    enabled: enabled && location !== null,
    retry: 2,
    placeholderData: (previousData) => previousData,
  });

  // Map server cards to Recommendation[]
  const cards: Recommendation[] = useMemo(() => {
    if (!query.data?.cards) return [];
    const deckMode = query.data.deckMode;
    return query.data.cards.map(card =>
      deckMode === 'nature'
        ? natureToRecommendation(card)
        : curatedToRecommendation(card)
    );
  }, [query.data]);

  return {
    cards,
    deckMode: query.data?.deckMode ?? 'curated',
    isLoading: query.isLoading,
    isFetching: query.isFetching,
    isFullBatchLoaded: !query.isLoading && !query.isFetching,
    error: query.error as Error | null,
    refetch: query.refetch,
  };
}
```

================================================================================
 COMMON MISTAKES TO AVOID
================================================================================

1. DON'T call discover-nature or generate-curated-experiences via HTTP from
   serve-deck. Import the logic directly or use shared modules. HTTP adds
   100-500ms latency per call.

2. DON'T forget placeholderData in useDeckCards. Without it, switching
   preferences shows an empty deck while the new cards load. This is THE
   key UX improvement.

3. DON'T remove useRecommendationsQuery entirely. Collaboration mode still
   needs it. Gate it with `enabled: isCollaborationMode`.

4. DON'T change the nature batch history Zustand store. It still works —
   just wire it to the unified deck's nature cards instead of the old
   useNatureCards hook.

5. DON'T invalidate 'deck-cards' AND 'nature-cards' AND 'curated-experiences'
   in AppHandlers. Only invalidate 'deck-cards' (for solo) and 'recommendations'
   (for collaboration). The old query keys are no longer used by the solo deck.
