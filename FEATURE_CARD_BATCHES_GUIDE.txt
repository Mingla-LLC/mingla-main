PART A — BUG FIX (do this first, it's one change)
File: app-mobile/src/contexts/RecommendationsContext.tsx
====================================================================

Find the useEffect that syncs recommendations. It starts with something
like (around line 427-432):

  useEffect(() => {
    if (!recommendationsData) {
      return;
    }
    // ... change detection logic ...
    if (recommendationsData.length > 0 || curatedRecommendations.length > 0) {
      setRecommendations(interleaveCards(recommendationsData, curatedRecommendations));
    }
  }, [...]);

MAKE THESE TWO CHANGES inside that effect:

CHANGE 1 — Replace the early return guard:
  OLD:
    if (!recommendationsData) {
      return;
    }
  NEW:
    if (!recommendationsData && curatedRecommendations.length === 0) {
      return;
    }
    const regularCards = recommendationsData ?? [];

CHANGE 2 — Replace every reference to `recommendationsData` inside this
effect with `regularCards`. There are typically 3-4 references:
  a) In the change detection: `prevRecs.length !== recommendationsData.length`
     → becomes: `prevRecs.length !== regularCards.length`
  b) `prevRecs.some((prev, idx) => prev.id !== recommendationsData[idx]?.id)`
     → becomes: `prevRecs.some((prev, idx) => prev.id !== regularCards[idx]?.id)`
  c) `previousRecommendationsRef.current = recommendationsData;`
     → becomes: `previousRecommendationsRef.current = regularCards;`
  d) The `interleaveCards()` call:
     `interleaveCards(recommendationsData, curatedRecommendations)`
     → becomes: `interleaveCards(regularCards, curatedRecommendations)`

TEST for the bug fix:
  1. In the app, select ONLY "Solo Adventure" intent, apply preferences
  2. Confirm the swipe deck shows curated plan cards instead of "No matches found"
  3. Also confirm that selecting regular categories (Nature, Drink) still works normally

====================================================================
COMMON MISTAKES TO AVOID
====================================================================

1. DON'T add batchSeed to the edge function request body.
   It's only a React Query cache discriminator key.
   The edge functions already shuffle independently.

2. DON'T put the .slice(0, 20) BEFORE the interleaveCards call.
   It must go AFTER, on the combined array:
   interleaveCards(regularCards, curatedRecommendations).slice(0, 20)

3. DON'T forget to pass batchSeed to ALL 5 useCuratedExperiences calls
   (solo-adventure, first-dates, romantic, friendly, group-fun).
   Missing even one means that type's cache won't bust on "Generate Another 20".

4. The existing `handleViewCardsAgain` function in SwipeableCards
   clears removedCards and resets currentCardIndex to 0.
   DON'T rewrite it — just call it as-is for "Review Previous Batch".
   For "Generate Another 20", call generateNextBatch() first, THEN
   handleViewCardsAgain() to reset the swipe state for the new deck.

5. DON'T change the "no matches" condition in SwipeableCards.
   The Part A bug fix in RecommendationsContext already prevents the
   no_matches error from blocking curated cards. The SwipeableCards
   "no matches" screen will only show when there are truly zero
   recommendations (which is correct behaviour).

6. DON'T touch collaboration mode code. The batchSeed is initialized
   to 0 and only changes on user action. Collaboration mode won't be
   affected since it has its own query path.
