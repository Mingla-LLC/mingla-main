# Specer Prompt: Category System Migration (13 Categories)

## Context

We're moving from 12 app categories to 13. The curated overhaul spec creates `_shared/seedingCategories.ts` as the single source of truth for category configs. This spec migrates ALL other functions and the mobile app to use the new 13-category system.

## The 13 Categories

| # | Category | Visible to User | Notes |
|---|----------|----------------|-------|
| 1 | Nature & Views | Yes | Was "Nature" |
| 2 | First Meet | Yes | Same |
| 3 | Picnic Park | Yes | Was "Picnic" |
| 4 | Drink | Yes | Same |
| 5 | Casual Eats | Yes | Same |
| 6 | Fine Dining | Yes | Same |
| 7 | Watch | Yes | Movie theaters only |
| 8 | Live Performance | Yes | NEW — was merged into Watch |
| 9 | Creative & Arts | Yes | Same |
| 10 | Play | Yes | Same |
| 11 | Wellness | Yes | Same |
| 12 | Flowers | Yes | Was part of "Groceries & Flowers" |
| 13 | Groceries | HIDDEN | Exists in system, never shown to users. Used for curated picnic stops. |

## What Changes

### Backend — Edge Functions

**1. `_shared/categoryPlaceTypes.ts`** — either replace entirely with imports from `seedingCategories.ts`, or update to 13 categories with new type mappings. The old `MINGLA_CATEGORY_PLACE_TYPES`, `GLOBAL_EXCLUDED_PLACE_TYPES`, `INTENT_EXCLUDED_PLACE_TYPES` all need updating:
- 13 categories with new includedTypes/excludedPrimaryTypes per the configs in `outputs/ARCHITECT_PROMPT_ADMIN_POOL_MANAGEMENT.md`
- GLOBAL_EXCLUDED_PLACE_TYPES: `['gym', 'fitness_center', 'dog_park']`
- INTENT_EXCLUDED_PLACE_TYPES: DELETE entirely (per-category excludedPrimaryTypes handle everything)
- Old searchMethod field: DELETE (no more Text Search distinction)
- ALL_CATEGORY_NAMES: update to 13
- resolveCategories(): must handle new names

**2. `generate-single-cards/index.ts`** — update to use 13 categories from shared source. Must handle Groceries (generate cards for it even though it's hidden — curated picnic needs grocery cards in card_pool).

**3. `discover-cards/index.ts`** — update scorePoolCards, filterByDateTime, and any category references to 13 categories. Groceries cards should NOT appear in regular discover-cards results (hidden category).

**4. `cardPoolService.ts`** — update `query_pool_cards` references, `poolCardToApiCard`, any category validation. Global exclusions: gym, fitness_center, dog_park.

**5. `query_pool_cards` SQL** — update `v_excluded_types` to `ARRAY['gym', 'fitness_center', 'dog_park']`. Ensure Groceries cards are excluded from regular single-card queries (only served via curated cards).

**6. All 5 card-serving functions** (discover-cards, discover-experiences, get-personalized-cards, get-holiday-cards, get-person-hero-cards) — update any hardcoded category lists or references.

### Mobile App

**7. Category list** — update from 12 to 12 visible + 1 hidden:
- PreferencesSheet: show 12 categories (not Groceries)
- Category pills on deck: 12 categories
- Any category filter/picker: 12 categories
- Add Live Performance and Flowers as new visible categories
- Remove "Groceries & Flowers" (split into Flowers visible + Groceries hidden)
- Rename "Nature" → "Nature & Views", "Picnic" → "Picnic Park" (if names changed)

**8. Category icons/colors** — Live Performance and Flowers need icons. Groceries & Flowers icon/color reassigned to Flowers.

**9. Curated experience type list** — separate spec handles this (FEATURE_CURATED_OVERHAUL_SPEC.md). But Friendly must be removed from mobile here too if not already handled.

## Groceries Hidden Category — Rules

- Groceries cards ARE generated by generate-single-cards (grocery stores exist in card_pool)
- Groceries cards are NOT shown in discover-cards regular results
- Groceries cards are NOT shown as a category pill
- Groceries is NOT in the PreferencesSheet
- Groceries places ARE used by generate-curated-experiences for picnic-dates stops
- Groceries category EXISTS in the shared category config (for seeding and curated generation)
