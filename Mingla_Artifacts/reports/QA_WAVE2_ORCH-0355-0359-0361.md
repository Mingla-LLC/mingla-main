### P3 — LOW: Dead style `sharedValues` in PersonBottomSheet

- **File**: `app-mobile/src/components/map/PersonBottomSheet.tsx:218`
- **Code**: `sharedValues: { fontSize: 12, color: '#6b7280', textAlign: 'center', marginTop: 4 },`
- **Issue**: This style is defined but no longer referenced in JSX. The "You both enjoy" text that used it was removed when the interests pills were added.
- **Impact**: None. Dead code. No visual or functional effect.
- **Fix**: Remove the `sharedValues` style entry.

### P4 — NOTE: Clean crash elimination pattern

The `.single()` → `.maybeSingle()` fix is textbook. The existing `!profile` guard in ViewFriendProfileScreen already handled the null case with a polished error screen. The implementor correctly identified that no UI change was needed — just preventing the crash so the existing handler could run.

### P4 — NOTE: Pin label approach is elegant

Using `card.title` split on arrow characters for curated cards extracts a meaningful first-stop name without needing a new field from the edge function. Truncation at 15 chars prevents overlap. The `numberOfLines={1}` prop ensures no multi-line wrapping.
