## Findings

### P3 — LOW: ReportUserModal `user.name` may show "User" if `selectedPerson` is nulled

- **File**: `app-mobile/src/components/map/DiscoverMap.tsx:527`
- **Code**: `name: selectedPerson?.displayName || 'User'`
- **Issue**: If the PersonBottomSheet closes (setting `selectedPerson` to null) while the ReportUserModal is open, the modal's header shows "Report User" with name "User" instead of the actual person's name.
- **Impact**: Cosmetic only. The `reportTargetUserId` is captured independently and the report is submitted correctly regardless.
- **Fix**: Capture `selectedPerson?.displayName` into a ref or state alongside `reportTargetUserId` when the report button is tapped.

### P4 — NOTE: Clean implementation pattern

The map report fix correctly mirrors the friend list report pattern: state → modal → service → result check. The implementor subtracted the broken code entirely before adding the replacement. This is how it should be done.
