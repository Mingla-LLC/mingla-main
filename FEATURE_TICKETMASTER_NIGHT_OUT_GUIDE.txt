OVERVIEW
--------
Replace the AI-generated Night Out tab (Google Places + OpenAI) with real
Ticketmaster events. Users browse real upcoming parties, filter by genre/date/
price, and buy tickets via an in-app browser. No Stripe. No Google Places.

Read the companion FEATURE_TICKETMASTER_NIGHT_OUT_SPEC.md for full type
definitions and architecture details. This guide tells you WHAT to do and in
WHAT ORDER.

STEP 3: CREATE THE TICKETMASTER EDGE FUNCTION
================================================================================

Create folder + file:
  supabase/functions/ticketmaster-events/index.ts

This function:
  1. Receives { location, radius?, keywords?, startDate?, endDate?, sort?, page?, size? }
  2. Builds a cache key from location (rounded to 1 decimal) + keywords + date
  3. Checks ticketmaster_events_cache for a non-expired entry
  4. On cache HIT: returns cached events immediately
  5. On cache MISS: calls Ticketmaster Discovery API v2, transforms response, caches, returns

--- KEY IMPLEMENTATION DETAILS ---

A) TICKETMASTER API CALL

   URL: https://app.ticketmaster.com/discovery/v2/events.json

   Query parameters:
     apikey          = Deno.env.get("TICKETMASTER_API_KEY")
     geoPoint        = "{lat},{lng}"   (from request body location)
     radius          = request radius or 50
     unit            = "km"
     segmentId       = "KZFzniwnSyZfZ7v7nJ"  (Music — hardcode this)
     keyword         = keywords.join(",")  (default: "afrobeats,amapiano,dancehall")
     startDateTime   = startDate or now() in "YYYY-MM-DDTHH:mm:ssZ" format
     endDateTime     = endDate or now()+30d in "YYYY-MM-DDTHH:mm:ssZ" format
     sort            = sort or "date,asc"
     size            = size or 20
     page            = page or 0

   Example full URL:
     https://app.ticketmaster.com/discovery/v2/events.json?apikey=KEY&geoPoint=40.7484,-73.9857&radius=50&unit=km&segmentId=KZFzniwnSyZfZ7v7nJ&keyword=afrobeats,amapiano,dancehall&startDateTime=2026-03-01T00:00:00Z&endDateTime=2026-03-31T23:59:59Z&sort=date,asc&size=20&page=0

B) RESPONSE TRANSFORMATION

   Ticketmaster returns: { _embedded: { events: [...] }, page: {...} }

   For each event in _embedded.events:

     id            = event.id
     eventName     = event.name
     artistName    = event._embedded?.attractions?.[0]?.name || "Various Artists"
     venueName     = event._embedded?.venues?.[0]?.name || "TBA"

     image         = Pick from event.images[]:
                     Prefer ratio "16_9" AND width >= 1024
                     Fallback: largest image by width
     images        = event.images.map(i => i.url) (deduplicated)

     priceMin      = event.priceRanges?.[0]?.min || null
     priceMax      = event.priceRanges?.[0]?.max || null
     priceCurrency = event.priceRanges?.[0]?.currency || "USD"
     price         = Format as "$25 - $75" or "TBA" if no priceRanges

     date          = Format event.dates.start.localDate as "Fri, Mar 15"
     time          = Format event.dates.start.localTime as "8:00 PM"
                     (handle missing localTime → "Time TBA")
     localDate     = event.dates.start.localDate
     localTime     = event.dates.start.localTime || ""
     dateTimeUTC   = event.dates.start.dateTime || ""

     location      = `${venue.city.name}, ${venue.state.stateCode}`
     address       = venue.address?.line1 || ""
     coordinates   = {
                       lat: parseFloat(venue.location?.latitude || "0"),
                       lng: parseFloat(venue.location?.longitude || "0")
                     }

     genre         = event.classifications?.[0]?.genre?.name || "Music"
     subGenre      = event.classifications?.[0]?.subGenre?.name || ""
     tags          = Derive from [genre, subGenre, segment.name].filter(Boolean)
                     Add "Live" if there are attractions

     ticketUrl     = event.url
     ticketStatus  = event.dates.status?.code || "onsale"
     seatMapUrl    = event.seatmap?.staticUrl || undefined

     distance      = Calculate haversine distance from user location to venue coords
                     Format as km rounded to 1 decimal

C) CACHE KEY FORMAT

   `geo:${lat.toFixed(1)}:${lng.toFixed(1)}:kw:${keywords.sort().join(",")}:d:${startDate.slice(0,10)}`

   Example: "geo:40.7:-74.0:kw:afrobeats,amapiano,dancehall:d:2026-03-01"

D) CACHE WRITE (fire-and-forget)

   UPSERT into ticketmaster_events_cache:
     cache_key = computed key
     events    = JSON array of transformed events
     total_results = page.totalElements
     fetched_at = now()
     expires_at = now() + 2 hours

   ON CONFLICT (cache_key) DO UPDATE SET events=..., fetched_at=..., expires_at=...

E) CLEANUP (opportunistic, on every cache write)

   DELETE FROM ticketmaster_events_cache WHERE expires_at < now();

F) HAVERSINE DISTANCE (no Google API needed)

   function haversineKm(lat1, lng1, lat2, lng2): number {
     const R = 6371; // Earth radius in km
     const dLat = (lat2 - lat1) * Math.PI / 180;
     const dLng = (lng2 - lng1) * Math.PI / 180;
     const a = Math.sin(dLat/2)**2 +
               Math.cos(lat1 * Math.PI / 180) * Math.cos(lat2 * Math.PI / 180) *
               Math.sin(dLng/2)**2;
     return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
   }

G) ERROR HANDLING

   - Ticketmaster 429 → check cache for ANY non-expired entry for this geo, return it
   - Ticketmaster 5xx → same fallback
   - Empty _embedded.events → return { events: [], meta: { totalResults: 0 } }
   - Missing env var → return 500 with clear message

H) CORS HEADERS (same pattern as all other edge functions)

   const corsHeaders = {
     "Access-Control-Allow-Origin": "*",
     "Access-Control-Allow-Headers":
       "authorization, x-client-info, apikey, content-type",
   };

6C) UPDATE DATE FILTER HELPER

  function getDateRange(filter: DateFilter): { startDate: string; endDate: string } {
    const now = new Date();
    const startOfDay = (d: Date) => {
      const copy = new Date(d);
      copy.setHours(0, 0, 0, 0);
      return copy;
    };
    const endOfDay = (d: Date) => {
      const copy = new Date(d);
      copy.setHours(23, 59, 59, 0);
      return copy;
    };

    switch (filter) {
      case "today":
        return { startDate: now.toISOString(), endDate: endOfDay(now).toISOString() };
      case "tomorrow": {
        const tmrw = new Date(now);
        tmrw.setDate(tmrw.getDate() + 1);
        return { startDate: startOfDay(tmrw).toISOString(), endDate: endOfDay(tmrw).toISOString() };
      }
      case "weekend": {
        // Find next Friday 6pm
        const dayOfWeek = now.getDay();
        const daysUntilFri = (5 - dayOfWeek + 7) % 7 || 7;
        const friday = new Date(now);
        friday.setDate(friday.getDate() + (dayOfWeek <= 5 && dayOfWeek > 0 ? daysUntilFri : 0));
        friday.setHours(18, 0, 0, 0);
        const sunday = new Date(friday);
        sunday.setDate(sunday.getDate() + 2);
        sunday.setHours(23, 59, 59, 0);
        // If it's already the weekend, start from now
        if (dayOfWeek === 0 || dayOfWeek === 6 || (dayOfWeek === 5 && now.getHours() >= 18)) {
          return { startDate: now.toISOString(), endDate: sunday.toISOString() };
        }
        return { startDate: friday.toISOString(), endDate: sunday.toISOString() };
      }
      case "next-week": {
        const monday = new Date(now);
        monday.setDate(monday.getDate() + (8 - now.getDay()) % 7);
        monday.setHours(0, 0, 0, 0);
        const nextSunday = new Date(monday);
        nextSunday.setDate(nextSunday.getDate() + 6);
        nextSunday.setHours(23, 59, 59, 0);
        return { startDate: monday.toISOString(), endDate: nextSunday.toISOString() };
      }
      case "month":
      case "any":
      default: {
        const monthLater = new Date(now);
        monthLater.setDate(monthLater.getDate() + 30);
        return { startDate: now.toISOString(), endDate: monthLater.toISOString() };
      }
    }
  }

================================================================================
COMMON MISTAKES TO AVOID
================================================================================

1. DON'T hardcode the Ticketmaster API key in the edge function source.
   Use Deno.env.get("TICKETMASTER_API_KEY").

2. DON'T forget that Ticketmaster geoPoint format is "lat,lng" NOT "lng,lat".

3. DON'T forget the segmentId parameter — without it you'll get non-music
   events (sports, theater, etc.).

4. DON'T parse prices from strings — use priceMin/priceMax numbers directly.
   The old code parsed "$10-$60" strings; the new code uses numeric fields.

5. DON'T forget to handle events with no priceRanges (some events don't list
   prices yet). Show "TBA" instead of "$0".

6. DON'T forget to handle events with no localTime — some events have date
   but no time. Show "Time TBA".

7. DON'T re-fetch on every price filter change — price is client-side only.
   Only genre and date trigger server re-fetch.

8. DON'T delete the old edge function until the new flow is fully tested.

9. DON'T forget CORS headers on the new edge function — mobile calls will
   fail silently without them.

10. DON'T open the ticket URL with Linking.openURL — use WebBrowser.openBrowserAsync
    for an in-app experience. Linking.openURL opens the system browser (leaves app).
